2. 사이트의 성능

직관적인가?

3. 브랜딩
웹 사이트의 첫 페이지가 당신의 비즈니스 성격을 반영하고 있는가?
브랜딩이 사이트의 모든 페이지에 나타나는가?
페이지 우측과 마찬가지로 페이지 상단의 주목할 만한 위치에 브랜딩이 표현되는가?
홈페이지로 링크를 거는 것처럼 브랜딩이 내비게이션의 한 부분처럼 작동하는가?
4. 내비게이션
사이트의 주요 내비게이션이 스크롤하기 전의 눈에 띄는 위치에 나타나는가?
사이트 내비게이션이 이미지 형식인가? 그렇다면 페이지 내에 텍스트 형식의 내비게이션도 고려해 본다.
이미지 태그에 ALT 속성을 사용하고 있는가?
내비게이션 툴에 자바스크립트를 사용하고 있다면 텍스트 링크도 함께 제공하고 있는가?
사이트에 데드 링크는 없는가? 데드 링크를 확인하면 즉시 제거한다.
사이트 전체를 내비게이션하지 않고 원하는 정보로 직접 찾아가는 사이트 맵이나 서치 엔진을 제공하고 있는가?
내비게이션 툴 및 시스템이 사이트 전체에 걸쳐 일관성 있는가?
사용자에게 앞뒤로 이동할 수 있도록 사이트 내의 정확한 위치 정보를 제공하고 있는가?
링크가 사용자를 어디로 데려갈지 정확하게 알려 주는가?
5. 이미지
콘텐츠와 관련 있는 맥락에서 이미지를 사용하고 있는가?
이미지 파일의 사이즈가 최적화되어 있는가?
큰 이미지를 작은 이미지로 자르는 것을 고려한다. 이렇게 하면 다운로드 속도를 빠르게 할 수 있다?
속도를 빠르게 할 수 있다.
이미지를 설명할 필요가 있다면 이미지에 대한 설명이 명확한가?
6. 애니메이션
연속적으로 회전하는 애니메이션 등으로 주의를 분산되는 것은 없는가?
플래시 애니메이션으로 로딩에 시간이 걸려서 불편한 것이 있는가?
7. 스폰서 및 광고
사이트에 배너로 인해 다운로드 속도가 느려져서 불편한 부분이 있는가?
배너의 위치가 페이지의 우측 상단이나 우측 하단에 있는가?
8. 콘텐츠
콘텐츠가 페이지나 사이트의 맥락과 일치하는가?
콘텐츠가 간략하고 정확한가? 사용자들은 적절한 정보만을 얻기 위해 콘텐츠를 흝어본다.
콘텐츠가 관련 있는 정보를 제공하는 문서나 웹 사이트로 크로스 링크되어 있는가?
콘텐츠를 분리하기 위해 수평선 규칙을 피한다. 페이지의 유용한 콘텐츠가 끝난다고 해석될 수 있기 때문이다.
9. 기술
사이트의 쌍방향적인 요소가 목표 고객이 사용하는 브라우저와 호환성이 있는가?
그렇지 않다면 사용자가 그것을 사용하기 위해 플러그인이나 소프트웨어를 다운로드받아야 하는가?
그런 기술의 사용이 중요하다면 사용자들에게 필요성과 중요성을 고지시켰는가?
10. 전체적인 인터페이스

레이아웃이 시각을 즐겁게 하기 위해 흰색의 여백을 많이 포함하고 있는가?
레이아웃과 색상과 관련해 사이트 전체적으로 인터페이스가 일관성 있는가?
상이한 플랫폼과 브라우저에서도 사이트가 동일하게 보이는가?
11. 피드백
고객의 입장에서 적절한 피드백 메커니즘을 제공하고 있는가?
예를 들어 사용자가 폼을 제출하면 수용 여부에 대해 알 수 있는가?
